# SportsPairing の他アプリ横展開：SPM 配布方針と GitHub Private 公開

## 概要
汎用ダブルスマッチメイクライブラリ SportsPairing を、複数アプリから再利用できるよう
**Swift Package Manager（SPM）+ GitHub Private リポジトリ + semver タグ** で配布する方針を確定し、
初版 `0.2.0` を公開した。

- リポジトリ: https://github.com/yoshitaka26/SportsPairing （Private）
- 初版タグ: `0.2.0`
- remote: `git@github.com:yoshitaka26/SportsPairing.git`（SSH）

## 背景
SportsPairing は PadeLovers から切り出した純ロジックの Swift Package。
多産戦略で他競技アプリへ横展開し、重み付けアルゴリズムを継続的に育てる前提がある。
「このリポジトリを他アプリで使う方法」を決める必要があり、SPM が適切か判断した。

## 原因・課題
- パッケージ化（`Package.swift` で `.library` 公開）は済んでいたが、**git 管理されていなかった**
  ため、リモート SPM 参照（URL + バージョン指定）ができない状態だった。
- 多アプリ横展開では「各アプリがどのバージョンを使うか」を固定できないと、
  ライブラリ改善のたびに全アプリが巻き込まれて壊れるリスクがある。

## 対応内容
1. 配布方法を検討し、リモート Git（GitHub Private）+ semver タグを本命に決定。
2. 事前確認：`.gitignore`（`.build`/`.DS_Store`/`xcuserdata` 除外済み）、`gh` 認証、git identity を確認。
3. `git init` → `main` ブランチ → 初回コミット（`.build`/`.DS_Store` が混入しないことを確認）。
4. `swift test` で 37 件全通過を確認してからタグを打つ運用とした（タグ＝動作保証点）。
5. `gh repo create SportsPairing --private --source=. --push` で Private 公開。
6. `git tag -a 0.2.0` を打って push。メモリ上の v0.2.0（多形式対応＋総当たり表生成）と整合させた。

## 設計判断
- **なぜ SPM か**：Apple 標準・追加ツール不要（Tuist/fastlane を使わない方針に合致）。
  外部依存ゼロの純ロジックライブラリなので SPM が最も素直。
- **なぜリモート Git（GitHub Private）か**：
  - ローカルパス参照（`.package(path:)`）は開発初期の高速イテレーションには良いが、
    同一マシン・配置依存で「多数アプリへ配布」には不向き。
  - バイナリ XCFramework は純 Swift ソースには過剰でソースデバッグ不可。非推奨。
  - リモート Git なら各アプリが **バージョンを固定**でき、改善を選択的に取り込める＝
    「継続的に育てる」前提と最も相性が良い。
- **タグ運用**：`swift test` 通過を確認してからタグを打ち、各アプリは安定版を参照する。
  破壊的変更時は major、後方互換の機能追加は minor を上げる（semver）。

## 技術的ポイント
- `.gitignore` で `/.build` を除外していても、`git init` 直後の `git add -A` で
  誤って成果物が入らないか **staged 一覧で必ず目視確認**してからコミットした。
- `platforms` は現状 `.iOS(.v17)` のみだが、Foundation 依存のみのため macOS ホストで
  `swift test` は問題なく通る（platforms 指定は最低デプロイ版の宣言であり、
  ホストビルドを排除しない）。
- Private リポジトリのため、各アプリの Xcode で参照する際は
  **GitHub アカウント認証（Settings > Accounts）または SSH キー**が必要。

## 各アプリ側の参照手順（メモ）
### Xcode の GUI から
1. File > Add Package Dependencies…
2. URL に `https://github.com/yoshitaka26/SportsPairing.git`
   （Private なので GitHub アカウントを Xcode の Accounts に追加しておく）
3. Dependency Rule を「Up to Next Major Version: 0.2.0」等で指定して追加。

### 他パッケージの Package.swift から参照する場合
```swift
dependencies: [
    .package(url: "https://github.com/yoshitaka26/SportsPairing.git", from: "0.2.0"),
],
targets: [
    .target(
        name: "YourApp",
        dependencies: [
            .product(name: "SportsPairing", targets: ["SportsPairing"])
        ]
    )
]
```

### 開発中に素早く回したい場合（任意）
ライブラリ自体を活発に改修中のアプリでは、一時的にローカルパス参照に切り替えると速い：
```swift
.package(path: "../SportsPairing")
```
安定したらタグ参照に戻す。

## 影響範囲
- SportsPairing リポジトリ：git 管理開始＋ GitHub Private 公開（破壊的変更なし、追加のみ）。
- 他アプリ：今後このパッケージを参照可能に。既存アプリへの即時影響はなし。

## 確認内容
- 実施：`swift test` 37 件全通過。staged ファイルに成果物混入なし。
  リモート push・タグ push 成功（`git tag -l` / `git remote -v` で確認）。
- 未実施：実際に別アプリへ組み込んでの取り込み確認（次に横展開する際に実施）。
  Private リポジトリの Xcode アカウント認証フローの動作確認も次回実機で。
